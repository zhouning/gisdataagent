# 团队协作与知识共享技能

## 概述

本技能支撑多用户协作场景下的知识积累与共享。通过空间记忆系统持久化分析上下文，
通过团队管理实现角色分工，通过数据资产共享促进跨成员复用，通过审计日志保障操作
可追溯。适用于多人协同的国土调查、规划编制和专题分析项目。

## 角色权限体系

### 三级角色模型
| 角色 | 权限范围 | 典型用户 |
|------|---------|---------|
| admin | 全部功能：用户管理、系统配置、所有管道访问 | 项目负责人、系统管理员 |
| analyst | 分析功能：Optimization + Governance + General 管道 | GIS 分析师、规划师 |
| viewer | 只读查询：仅 General 管道的查询功能 | 领导查阅、外部评审 |

### 权限控制要点
- RBAC 检查在语义路由后、管道调度前执行
- viewer 尝试访问 Governance/Optimization 管道时返回权限不足提示
- 管理员可通过 `/api/admin/users/{username}/role` 调整角色
- 数据库层面通过 `SET app.current_user` 支持行级安全（RLS）

## 空间记忆系统

### 记忆类型

#### region（关注区域）
- 用途：记录用户关注的地理范围和区域特征
- 示例：`"关注斑竹村耕地整治区域，面积约320亩"`、`"重点监测岷江流域河道两侧500米缓冲区"`
- 场景：后续分析自动聚焦到关注区域，减少重复指定范围

#### viz_preference（可视化偏好）
- 用途：记录用户的地图样式和展示偏好
- 示例：`"偏好热力图展示人口密度"`、`"用地类型使用标准三调配色方案"`
- 场景：生成可视化时自动应用偏好设置

#### analysis_result（分析结论）
- 用途：持久化关键分析发现，供后续引用和对比
- 示例：`"斑竹村FFI=0.45，碎片化程度中等，建议整合零散地块"`
- 场景：跨会话保留分析结论，支持时序对比（上次 FFI=0.52，本次下降到 0.45）

#### custom（自定义上下文）
- 用途：记录用户身份、专业背景和分析视角
- 示例：`"林业规划师视角，关注生态红线和林地保有量"`
- 场景：影响 Agent 的分析侧重点和报告措辞

### 记忆操作

#### 存储记忆
- `save_memory`：保存一条空间记忆
  - 参数：content（内容）、memory_type（类型）、tags（标签列表）
  - 自动关联当前用户 ID 和时间戳
  - 建议：只保存有长期价值的结论和偏好，避免琐碎信息

#### 检索记忆
- `recall_memories`：按条件检索记忆
  - 按类型过滤：`memory_type='analysis_result'`
  - 按关键词搜索：`keyword='碎片化'`
  - 按时间范围：`after='2026-01-01'`
  - 返回按相关度和时间排序的记忆列表

#### 记忆注入
- 分析视角（`analysis_perspective`）自动注入到 Agent 提示词
- 通过 UserSettings 界面设置，存储在用户配置中
- 每次对话开始时读取并附加到系统提示

### 记忆管理最佳实践
- 每次重要分析完成后保存 1-2 条核心结论
- 定期清理过时的 region 记忆（区域变更后）
- analysis_result 附带数据日期标签，便于时序追踪
- 避免保存中间过程数据，只保存最终结论和决策依据

## 数据资产共享

### 共享流程
1. 完成数据处理并验证质量（完整性、准确性）
2. `register_data_asset` 注册到数据目录（填写描述和来源）
3. `tag_data_asset` 添加主题标签（便于团队成员发现）
4. `share_data_asset` 指定共享对象（用户或团队）
5. 通知团队成员数据已就绪

### 共享注意事项
- 共享前确认数据不含敏感信息（个人隐私、涉密坐标）
- 添加清晰的数据描述：来源、时间范围、坐标系、质量说明
- 大型数据集建议附带数据字典或字段说明
- 共享操作自动记录审计日志，可追溯

## 审计日志

### 日志查询
- `query_audit_log`：查询操作审计记录
  - 按用户过滤：`username='zhangsan'`
  - 按操作类型：`action='import_data'`、`action='share_asset'`
  - 按时间范围：`start_time`、`end_time`
  - 按状态：`status='success'` 或 `status='failed'`

### 审计覆盖范围
- 数据操作：导入、导出、删除、共享
- 分析操作：管道执行、报告生成
- 管理操作：用户创建、角色变更、系统配置
- 认证事件：登录、登出、密码修改

## 协作工作流示例

### 多人国土调查项目
1. admin 创建项目团队，分配 analyst 和 viewer 角色
2. analyst-A 导入基础地理数据，注册到数据目录并共享
3. analyst-B 检索共享数据，执行叠加分析，保存分析结论到记忆
4. analyst-A 通过 `recall_memories` 查看 B 的分析结论，继续深化
5. viewer 查阅最终分析结果和生成的报告
6. admin 通过审计日志回顾项目全流程操作记录

## 常见问题与解决

- 记忆污染：保存过多琐碎记忆导致检索噪声大，定期清理低价值记忆
- 共享权限不足：确认当前用户角色为 analyst 或 admin
- 审计日志量大：使用多条件组合过滤缩小范围，避免全量查询
- 记忆跨用户不可见：记忆默认用户隔离，需通过数据共享机制传递结论
- 角色变更不生效：角色变更在下次登录时生效，当前会话保持原角色
