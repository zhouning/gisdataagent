# 地理编码与距离计算技能

## 核心能力

1. **正向编码**: `batch_geocode` 地址 → 坐标（批量）
2. **逆向编码**: `reverse_geocode` 坐标 → 地址
3. **驾车距离**: `calculate_driving_distance` 两点间驾车距离和时间
4. **POI 搜索**: `search_poi` 周边兴趣点搜索
5. **行政区划**: `get_admin_boundary` 获取行政区划边界
6. **人口数据**: `download_population` WorldPop 100m 栅格人口数据

## 正向地理编码

### 精度等级
| 等级 | 说明 | 适用场景 |
|------|------|----------|
| 门牌号级 | 精确到建筑物 | 快递配送、精确定位 |
| 道路级 | 精确到道路 | 路线规划 |
| 区县级 | 精确到区/县 | 区域统计 |
| 城市级 | 精确到城市 | 粗略定位 |

### 地址预处理要求
- 去除多余空格和特殊字符
- 补全省/市前缀（"朝阳区" → "北京市朝阳区"）
- 统一格式（"XX路XX号" 标准化）
- 拆分复合地址（"A路B号C栋" → 分层编码）

### 批量编码注意事项
- API 限流: 高德 API 默认 30 QPS，大批量需分批
- 失败重试: 网络超时自动重试 3 次
- 置信度过滤: 建议过滤 confidence < 0.7 的结果
- 结果坐标系: 高德返回 GCJ-02，需转换为 WGS84 使用

## 逆向地理编码

- 输入: 经纬度坐标
- 输出: 结构化地址（省/市/区/街道/门牌号）
- 注意坐标系: 输入必须为 GCJ-02（高德 API）或先转换
- 适用场景: GPS 轨迹点地址解析、坐标标注

## 驾车距离计算

### 与直线距离的区别
| 指标 | 直线距离 | 驾车距离 |
|------|---------|---------|
| 计算方式 | Haversine 公式 | 路网规划 |
| 速度 | 毫秒级 | 需调用 API |
| 精度 | 粗略 | 考虑道路实际情况 |
| 适用 | 初筛、大批量 | 精确分析、少量 |

### 使用建议
- 大批量（>1000 对）先用直线距离初筛，再对候选对计算驾车距离
- 驾车距离受实时路况影响，结果为参考值
- 返回值包含距离（米）和预计时间（秒）

## POI 搜索

- `search_poi(keyword, city, radius)`: 按关键词搜索周边 POI
- 支持分类搜索: 学校、医院、超市、加油站等
- 半径范围: 建议 1000-5000m
- 结果包含: 名称、地址、坐标、类型、距离

## 行政区划查询

- `get_admin_boundary(name, level)`: 获取行政区划边界 GeoJSON
- level: province/city/district/street
- 支持子区划获取: `subdistrict=1` 获取下一级
- 适用: 区域统计、行政区划底图

## 分析工作流

1. **数据预处理**: 清洗地址字段，标准化格式
2. **批量编码**: `batch_geocode(addresses)` → 获取坐标
3. **置信度过滤**: 过滤低置信度结果（< 0.7）
4. **结果验证**: 抽样检查编码结果是否合理
5. **坐标转换**: GCJ-02 → WGS84（如需与其他数据叠加）
6. **可视化**: `visualize_interactive_map` 展示编码结果

## 常见问题与解决方案

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 编码结果偏移 | 坐标系混用（GCJ-02 vs WGS84） | 统一转换坐标系 |
| 大量编码失败 | 地址格式不规范 | 预处理标准化地址 |
| 编码到错误位置 | 地址歧义（同名地点） | 补全省市前缀消歧 |
| API 超限 | 请求频率过高 | 分批处理，控制 QPS |
| 逆向编码无结果 | 坐标在海洋/无人区 | 检查坐标范围合理性 |
