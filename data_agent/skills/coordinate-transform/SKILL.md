# 坐标系转换与验证技能

## 核心能力

1. **坐标系识别**: `describe_geodataframe` 检查当前 CRS
2. **坐标转换**: `reproject_spatial_data` 执行坐标系转换
3. **特征工程**: `engineer_spatial_features` 在投影坐标系下计算面积/周长

## 中国常用坐标系

### 地理坐标系（经纬度）

| 坐标系 | EPSG | 说明 | 使用场景 |
|--------|------|------|----------|
| CGCS2000 | 4490 | 中国国家大地坐标系 2000 | 国土调查、官方数据 |
| WGS84 | 4326 | GPS 全球坐标系 | GPS 数据、Web 地图 |
| GCJ-02 | — | 高德/腾讯加密坐标（火星坐标） | 高德/腾讯地图 API |
| BD-09 | — | 百度加密坐标 | 百度地图 API |

### CGCS2000 vs WGS84
- 差异在厘米级别（<2cm），绝大多数应用场景可互换
- 国土调查数据标准为 CGCS2000
- 严格场景（高精度测量）需注明坐标系来源

### GCJ-02 偏移警告
- 相对 WGS84 偏移约 **300-500m**（非线性偏移）
- **绝不能**将 GCJ-02 坐标当作 WGS84 使用
- 高德/腾讯地图 API 返回的坐标均为 GCJ-02
- 转换需要专门的纠偏算法（非简单平移）

### BD-09 偏移
- 在 GCJ-02 基础上再次加密
- 百度地图 API 返回的坐标为 BD-09
- 需先转 GCJ-02 再转 WGS84（两步转换）

## 投影坐标系（平面米制）

### CGCS2000 高斯-克吕格投影
- **3° 带**（推荐）: 适用于省级及以下精确量测
  - 带号 = round(经度 / 3)
  - 中央经线 = 带号 × 3°
  - 例: 经度 117° → 39 带 → 中央经线 117°E → EPSG:4547
- **6° 带**: 适用于全国性概览
  - 带号 = floor((经度 + 6) / 6)
  - 中央经线 = 带号 × 6° - 3°

### UTM 投影
- 国际通用 6° 分带
- 中国范围: UTM Zone 43N ~ 53N (EPSG:32643 ~ 32653)
- 适用于国际合作项目

## 坐标系选择原则

| 场景 | 推荐坐标系 | 原因 |
|------|-----------|------|
| 数据存储 | EPSG:4490 或 4326 | 通用性好，精度无损 |
| 面积计算 | 当地高斯 3° 带 | 平面坐标，面积准确 |
| 距离计算 | 当地高斯 3° 带或 UTM | 平面坐标，距离准确 |
| Web 地图展示 | EPSG:4326 | Leaflet/Folium 标准 |
| 国土调查数据 | EPSG:4490 | 国家标准 |
| 缓冲区分析 | 投影坐标系 | 地理坐标系下缓冲区为椭圆 |
| 数据融合/叠加 | 统一为 4326 或 4490 | 避免坐标系混用 |

## 分析工作流

1. **识别源坐标系**: `describe_geodataframe` 查看 CRS 信息
2. **确定目标坐标系**: 根据分析需求选择（见上表）
3. **执行转换**: `reproject_spatial_data(target_crs='EPSG:4326')`
4. **验证结果**: 检查坐标范围是否合理（中国经度 73°-135°，纬度 18°-54°）
5. **面积一致性**: 转换前后面积应基本一致（投影→投影）

## 常见问题与解决方案

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 数据偏移 300-500m | GCJ-02 数据误当 WGS84 | 使用纠偏算法转换 |
| 面积计算结果为平方度 | 在地理坐标系下计算面积 | 先投影到高斯带 |
| 缓冲区形状异常 | 在地理坐标系下做缓冲区 | 先投影到米制坐标系 |
| 高斯带选错 | 数据跨带 | 选择中央经线最近的带 |
| 叠加分析错位 | 两层坐标系不一致 | 统一转换到同一坐标系 |
| CRS 显示为 None | 数据缺少坐标系信息 | 根据坐标范围推断并设置 |
