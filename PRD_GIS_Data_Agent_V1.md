# GIS Data Agent V1 — 产品需求文档 (PRD)

> **文档版本**: V1.0
> **日期**: 2026-02-22
> **产品名称**: GIS Data Agent（空间数据智能体）
> **技术形态**: Web 应用（浏览器访问）
> **目标市场**: 中国市场优先，兼顾国际化

---

## 一、Problem Statement（问题陈述）

**在空间数据分析领域存在一个断层**：一边是强大但门槛极高的专业 GIS 工具（ArcGIS、QGIS），需要多年学习才能上手；另一边是普及但不具备空间分析能力的通用 AI 工具（ChatGPT、Julius AI），无法处理地图、坐标、区域分析等空间问题。

**受影响的人群**：大量数据分析师、企业决策者、研究人员需要从空间维度获取洞察（选址在哪里？客户分布如何？哪个区域增长最快？），但他们不会用 GIS 软件，也请不起 GIS 专家。

**不解决的后果**：这些用户要么放弃空间分析维度（损失决策质量），要么花费数周学习 GIS 工具（效率低下），要么雇佣专人完成（成本高昂）。根据市场数据，垂直领域 AI Agent CAGR 达 62.7%，GIS + Data Agent 交叉区域目前几乎无竞品，这是一个明确的产品机会窗口。

---

## 二、Goals（产品目标）

| # | 目标 | 衡量标准 |
|---|------|---------|
| G1 | **让非 GIS 用户能用自然语言完成专业级空间分析** | 用户从上传数据到获得地图可视化 < 2 分钟；无需学习任何 GIS 概念 |
| G2 | **成为"GIS Data Agent"品类的定义者** | 上线 6 个月内成为该关键词搜索排名 Top 3 |
| G3 | **建立可持续的商业模式** | 免费版获客 > 10,000 用户；企业版签约 ≥ 3 家付费客户（上线 6 个月内） |
| G4 | **覆盖通用空间分析场景** | 支持选址分析、区域对比、热力图、缓冲区分析等 10+ 核心空间分析方法 |
| G5 | **满足数据安全与合规要求** | 通过数据安全法和测绘法合规审查，空间数据处理符合国内法规 |

---

## 三、Non-Goals（非目标 / 明确不做的事）

| # | 非目标 | 理由 |
|---|--------|------|
| NG1 | **不做专业级 GIS 编辑工具**（如矢量编辑、拓扑检查、坐标系高级转换） | 不与 ArcGIS/QGIS 正面竞争，定位互补而非替代 |
| NG2 | **V1 不做遥感影像分析** | 遥感影像处理复杂度极高，且需要大量计算资源，不适合 V1 |
| NG3 | **V1 不做移动端 App** | 先验证 Web 端核心价值，移动端延后 |
| NG4 | **V1 不做多 Agent 协作编排** | 参考 Frontier 模式，预留架构但不作为 V1 交付 |
| NG5 | **V1 不做消息 Bot 入口**（微信/飞书/钉钉） | 参考 OpenClaw 模式，V2 可扩展；V1 先做好 Web 端核心体验 |
| NG6 | **不做通用数据分析**（与 Julius AI/ChatGPT 竞争） | 专注空间分析差异化，通用统计分析仅作为空间分析的辅助模块 |

## 四、User Stories（用户故事）

### 4.1 Persona A：数据分析师（非 GIS 背景）— 主力用户

> **背景**：某零售企业的数据分析师小王，擅长 Excel/SQL/Python 数据分析，但从未用过 GIS 软件。领导要求做一份"门店选址空间分析报告"。

| ID | 用户故事 | 优先级 |
|----|---------|:------:|
| A1 | 作为数据分析师，我想**上传一份包含门店地址的 Excel 文件**，系统自动将地址转换为坐标并标注在地图上，这样我不需要学习地理编码就能看到门店分布。 | P0 |
| A2 | 作为数据分析师，我想**用自然语言提问**"这些门店哪些聚集在一起？哪些是孤立的？"，系统自动执行空间聚类分析并在地图上用颜色区分，这样我不需要知道 DBSCAN 算法就能看到聚类结果。 | P0 |
| A3 | 作为数据分析师，我想**对话追问**"在聚类 A 周围 1 公里内有多少竞品门店？"，系统自动做缓冲区分析并叠加竞品数据，这样我可以像跟分析师同事对话一样逐步深入分析。 | P0 |
| A4 | 作为数据分析师，我想**一键生成分析报告**（包含地图截图 + 数据图表 + 文字洞察），这样我可以直接发给领导而不需要手动截图拼 PPT。 | P1 |
| A5 | 作为数据分析师，我想让系统**记住我常分析的区域和偏好**（如"我通常分析华东区域，用深色底图"），这样下次打开不需要重新设置。 | P1 |

### 4.2 Persona B：GIS 从业者

> **背景**：GIS 工程师小李，日常使用 ArcGIS/QGIS，但很多简单的可视化和分析任务太繁琐。

| ID | 用户故事 | 优先级 |
|----|---------|:------:|
| B1 | 作为 GIS 从业者，我想**上传 GeoJSON/Shapefile 文件**直接在地图上渲染，并用自然语言描述想做的分析（如"对这些面数据做空间自相关分析"），这样比在 ArcGIS 中点十几步菜单快得多。 | P0 |
| B2 | 作为 GIS 从业者，我想**查看 Agent 执行的分析代码**（Python/SQL），确认分析逻辑正确后再采信结果，这样我可以信任 Agent 的输出。 | P1 |
| B3 | 作为 GIS 从业者，我想把 Agent 生成的分析流程**导出为 Python 脚本**，集成到我的自动化工作流中。 | P2 |

### 4.3 Persona C：企业决策者 / 管理层

> **背景**：连锁品牌区域总监张总，需要看空间维度的经营分析但自己不做数据处理。

| ID | 用户故事 | 优先级 |
|----|---------|:------:|
| C1 | 作为企业决策者，我想**用自然语言提问**"我们华东区域哪些城市的门店密度还有提升空间？"，直接得到地图 + 数据支撑的回答，这样我不需要等分析师做报告。 | P0 |
| C2 | 作为企业决策者，我想看到**多维度区域对比仪表盘**（人口、消费力、竞品、我方门店），直观了解各区域机会。 | P1 |
| C3 | 作为企业决策者，我想**分享分析结果链接**给团队成员查看，不需要他们也注册账号。 | P1 |

### 4.4 Persona D：学生 / 研究者

> **背景**：城市规划专业研究生小陈，论文需要做空间数据分析但 GIS 水平有限。

| ID | 用户故事 | 优先级 |
|----|---------|:------:|
| D1 | 作为研究者，我想**免费使用基础空间分析功能**来完成课题研究，这样我不需要花钱购买 ArcGIS 许可。 | P0 |
| D2 | 作为研究者，我想让 Agent **解释每步分析的方法和依据**（如"使用了 Moran's I 指数来衡量空间自相关"），这样我可以在论文中正确引用方法。 | P1 |
| D3 | 作为研究者，我想**导出分析过程的完整日志**，确保研究的可复现性。 | P2 |

## 五、Requirements（功能需求）

### 5.1 P0 — Must-Have（必须交付）

#### 5.1.1 自然语言交互引擎

**描述**：用户通过对话框输入自然语言，Agent 理解空间分析意图，选择合适的 GIS 工具执行，并返回结果。

**验收标准**：
- [ ] 支持中文和英文自然语言输入
- [ ] 能正确识别空间分析意图（地图可视化、缓冲区分析、空间查询、聚类分析等），意图识别准确率 ≥ 85%
- [ ] 支持多轮对话，能理解上下文（如"把上面的结果按区域分组"中的"上面的结果"）
- [ ] 对话过程中展示实时状态指示（"正在进行地理编码..."、"正在生成热力图..."）
- [ ] 当意图不明确时，Agent 主动追问而非猜测执行
- [ ] 每次分析结果附带简要的方法说明（"使用了 XX 方法，基于 XX 数据"）

#### 5.1.2 数据导入与处理

**描述**：支持多种格式的空间/非空间数据导入，自动识别空间字段并转换。

**验收标准**：
- [ ] 支持文件上传：CSV、Excel (.xlsx)、GeoJSON、Shapefile (.zip)、KML
- [ ] 单文件大小限制 ≤ 100MB，单次上传支持多文件
- [ ] 自动识别地址列并批量地理编码（地址 → 经纬度），支持中文地址
- [ ] 自动识别经纬度列（兼容多种列名：lat/lng、纬度/经度、latitude/longitude 等）
- [ ] 数据预览：上传后展示表格预览 + 地图预览，用户确认后进入分析
- [ ] 数据清洗提示：自动检测空值、异常坐标（如经纬度=0）并提醒用户
- [ ] 支持连接 PostgreSQL/PostGIS 数据库进行空间查询

#### 5.1.3 地图可视化引擎

**描述**：交互式 Web 地图，支持多种可视化方式，用户可通过自然语言控制地图。

**验收标准**：
- [ ] 底图支持切换：高德地图、天地图（国内合规）、OpenStreetMap
- [ ] 可视化类型：
  - 点标注（支持自定义图标、大小、颜色映射）
  - 热力图
  - 聚类图（点聚合）
  - 等值区域图（Choropleth）
  - 气泡图（大小+颜色双变量）
- [ ] 交互操作：缩放、平移、点击查看详情（Tooltip/Popup）、图例
- [ ] 图层管理：支持叠加多个数据图层，可控制显隐和顺序
- [ ] 自然语言控制地图：用户说"放大到北京市朝阳区"，地图自动定位
- [ ] 地图截图/导出：支持导出为 PNG 图片

#### 5.1.4 核心空间分析能力

**描述**：Agent 可执行的空间分析方法集合，通过自然语言触发。

**验收标准**：
- [ ] **缓冲区分析**：给定点/线/面，生成指定半径的缓冲区
- [ ] **空间查询**：点在面内判断、距离查询（"找出 A 点 3km 内的所有 B"）
- [ ] **空间聚类**：DBSCAN / K-Means 空间聚类
- [ ] **热力分析**：基于点密度或属性值生成热力面
- [ ] **区域统计**：按行政区/自定义区域汇总统计（计数、求和、均值）
- [ ] **距离与路径**：两点间直线距离、驾车距离（调用地图API）
- [ ] **地理编码/逆地理编码**：地址↔坐标双向转换
- [ ] 每项分析结果以 **地图 + 数据表格 + 文字洞察** 三位一体呈现

#### 5.1.5 用户系统与安全基础

**描述**：用户注册、登录、数据隔离、基本权限控制。

**验收标准**：
- [ ] 支持邮箱注册/登录 + 手机号注册/登录
- [ ] 用户数据严格隔离，A 用户无法看到 B 用户的数据和分析
- [ ] 上传数据加密存储，传输使用 HTTPS
- [ ] 用户可删除自己的数据和分析记录
- [ ] 符合《数据安全法》和《个人信息保护法》基本要求
- [ ] 空间数据处理不涉及违反《测绘法》的行为（如不提供高精度坐标纠偏服务）

---

### 5.2 P1 — Nice-to-Have（重要但可延期）

#### 5.2.1 空间记忆系统

**描述**：Agent 记住用户的分析偏好、常用区域、历史洞察（借鉴 OpenClaw 持久记忆模式）。

**验收标准**：
- [ ] 记住用户常用的分析区域（如"华东区域"对应的行政区列表）
- [ ] 记住用户的可视化偏好（底图风格、配色方案、标注方式）
- [ ] 可引用历史分析结果（"上次分析的三个候选选址分别在哪里？"）
- [ ] 用户可查看和管理记忆内容，可手动删除特定记忆

#### 5.2.2 分析报告生成

**描述**：将对话式分析过程整理为结构化报告。

**验收标准**：
- [ ] 一键生成 PDF/Word 格式报告
- [ ] 报告包含：标题、分析背景、地图截图、数据图表、关键发现、方法说明
- [ ] 报告支持自定义模板（标题、Logo、页眉页脚）
- [ ] 生成的报告可通过链接分享（只读，无需登录即可查看）

#### 5.2.3 分析过程可解释与可回溯

**描述**：让用户理解 Agent 做了什么、为什么这样做（借鉴 Frontier 评估系统理念）。

**验收标准**：
- [ ] 每步分析展示方法说明（如"使用 DBSCAN 聚类，eps=500m, min_samples=5"）
- [ ] 分析步骤可展开查看执行的代码（Python/SQL）
- [ ] 每步分析可单独重新执行（修改参数后重跑）
- [ ] 地理编码结果标注置信度（高/中/低）

#### 5.2.4 公开地理数据接入

**描述**：Agent 可主动获取公开地理数据来丰富分析。

**验收标准**：
- [ ] 接入 POI 数据（高德/百度 POI 搜索 API）
- [ ] 接入行政区划边界数据（省/市/区/街道）
- [ ] 接入人口统计数据（区/县级）
- [ ] 数据来源在分析结果中明确标注

#### 5.2.5 协作与分享

**描述**：分析结果的分享功能（免费版支持只读分享；企业版支持团队协作）。

**验收标准**：
- [ ] 【免费版】分析项目可通过链接分享给他人查看（只读）
- [ ] 【免费版】分享链接可设置密码和有效期
- [ ] 【企业版】团队成员可共同编辑分析项目
- [ ] 【企业版】支持评论功能：查看者可在地图上标注点并留言
- [ ] 【企业版】空间数据资产可在团队内共享复用

#### 5.2.6 Token 消费管理

**描述**：用户可监控和控制 LLM API 消费（借鉴 OpenClaw 费用失控教训）。

**验收标准**：
- [ ] 用户仪表盘展示本月 Token 消费量和估算费用
- [ ] 支持设置月度消费上限，到达上限后暂停服务并提醒
- [ ] 免费版有每日分析次数限制（如 20 次/天）

## 八、定价策略

采用 **"免费版 + 企业版"双轨模式**：免费版功能大方以最大化获客，企业版对标 OpenAI Frontier 模式提供定制化商业服务。

| 版本 | 价格 | 定位 | 核心权益 | 限制 |
|------|------|------|---------|------|
| **免费版** | ¥0 | 个人用户、学生、研究者、小团队 | 全部空间分析方法 + 全部可视化类型 + 报告生成 + 公开数据 API + 空间记忆 | 单文件 50MB；每日分析 20 次；无数据库连接；无私有化部署；数据保留 90 天 |
| **企业版** | 联系销售 | 企业 GIS 团队、零售/地产/物流/政府 | 免费版全部功能 + 无限制使用 + 数据库连接 + 空间语义层 + Agent 权限与审计 + 私有化部署 + 专属 SLA + 团队协作 + 优先技术支持 | 按需定制 |

**策略逻辑**：

```
┌────────────────────────────────────────────────────────────┐
│                                                            │
│  免费版 (获客飞轮)              企业版 (商业变现)            │
│                                                            │
│  • 功能大方，不阉割核心能力      • 对标 Frontier 模式       │
│  • 用口碑和社区效应获客          • 联系销售，按企业需求定制  │
│  • 培养用户习惯和数据沉淀        • 核心增值:                │
│  • 个人/学生/研究者免费用        │  - 私有化部署(数据不出境) │
│  • 限制: 用量上限 + 无企业功能   │  - 空间语义层(组织级)    │
│                                 │  - 权限审计(合规)        │
│         ──────────────────►     │  - 数据库直连(生产级)    │
│         当用户在企业中推广时      │  - SLA + 技术支持       │
│         自然转化为企业版需求      │                         │
│                                                            │
└────────────────────────────────────────────────────────────┘
```

> **为什么不做中间版本（Pro/Team）**：
> 1. 中间版本定价（¥99-199/月）在空间分析领域吸引力有限 — 个人用户嫌贵，企业用户嫌功能不够
> 2. 免费版功能越大方，口碑传播越快，企业版转化漏斗越宽
> 3. 简化产品线，集中精力做好两个极端：**最好的免费体验** 和 **最专业的企业服务**
> 4. 对标 Frontier "联系销售"模式，企业客单价远高于 SaaS 订阅

## 十一、设计原则（从标杆产品提炼）

> 以下五条原则贯穿整个产品设计，来源于 OpenClaw 和 OpenAI Frontier 标杆分析。

### 原则 1：界面消失原则

> 最好的 GIS 界面是没有 GIS 界面。用户不需要看到图层面板、坐标系选择器、投影参数。

**实践**：所有 GIS 操作通过自然语言完成，专业参数自动推断，仅在需要时通过简单选项确认。

### 原则 2：渐进式披露

> 简单任务零配置，复杂任务可深入。

**实践**：
- Level 1："帮我看看门店分布" → 自动出图
- Level 2："用 DBSCAN 聚类，eps=500m" → 参数级控制
- Level 3："查看执行代码" → 代码级透明

### 原则 3：信任可验证

> 空间分析结果必须可验证、可解释、可复现。

**实践**：每次分析附带数据来源声明、方法说明、置信度标注。分析过程可回溯重放。

### 原则 4：空间记忆

> 系统应像有经验的分析师一样，记住用户的业务上下文和分析偏好。

**实践**：记住常用区域、可视化偏好、业务术语映射、历史分析结论。

### 原则 5：安全纵深

> 空间数据涉及国家安全和商业机密，安全不是可选项。

**实践**：沙箱执行环境、空间数据分级管理、操作日志留痕、合规底图和坐标系。
