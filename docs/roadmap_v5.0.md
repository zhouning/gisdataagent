## v4.1 — 稳定性增强 + 用户体验精打磨

**目标**: 在 v4.0 基础上查漏补缺，提升日常使用体验。
**周期**: 2-3 周

### 4.1.1 分析会话持久化 ⭐ 高优先级

**问题**: 当前 `InMemorySessionService`，Chainlit 重启后会话丢失，用户刷新页面无法恢复对话。

**方案**:
- 迁移到 `DatabaseSessionService` (PostgreSQL)
- 用户刷新页面可恢复对话上下文
- 支持会话历史列表（侧边栏）

**影响范围**: `app.py`, 新迁移脚本 `017_create_sessions.sql`

### 4.1.2 管道执行进度可视化 ⭐ 高优先级

**问题**: 用户只能看到折叠的"思考过程"，不清楚当前执行阶段。

**方案**:
- 前端新增 `PipelineProgress` 组件
- 利用 Chainlit step 事件显示阶段进度：`阶段 2/6: 正在数据质量审计... (3.2s)`
- 时间轴 UI 展示已完成/进行中/待执行步骤

**影响范围**: `ChatPanel.tsx`, `layout.css`

### 4.1.3 数据预览增强

**问题**: CSV 标签页只能查看平面表格，空间数据无法直观预览。

**方案**:
- DataPanel 增加 GeoJSON 缩略图预览（canvas 绘制 bbox + 要素轮廓）
- Shapefile 字段统计摘要（行数、列类型、空值率）
- 大文件懒加载（前 1000 行 + 虚拟滚动）

**影响范围**: `DataPanel.tsx`

### 4.1.4 国际化 (i18n) 基础框架

**问题**: UI 文本硬编码中文，无法服务英文用户。

**方案**:
- 引入 `react-i18next`
- 抽取现有中文为 `locales/zh.json`
- 新增 `locales/en.json`
- 语言切换按钮（header 区域）

**影响范围**: 所有前端组件, `package.json`

### 4.1.5 错误恢复与重试 ⭐ 高优先级

**问题**: 分析失败后用户需要重新描述整个请求。

**方案**:
- ChatPanel 增加"重试"按钮
- 携带原始消息 + 上传文件重新触发管道
- 错误类型分类：可重试（网络/超时）vs 不可重试（数据错误）

**影响范围**: `ChatPanel.tsx`, `app.py`

## v5.0 — 多模态 + 3D + MCP 生态

**目标**: 打开生态壁垒，支持多模态输入输出，建立工具市场。
**周期**: 1-2 月

### 5.1 MCP 工具市场 (Model Context Protocol) ⭐ 战略级

这是最具战略意义的功能，直接打开平台生态壁垒。

```
┌──────────────────────────────────────────────┐
│                MCP Tool Hub                   │
│  ┌──────────┐ ┌──────────┐ ┌──────────────┐  │
│  │ 内置工具  │ │ 社区工具  │ │ 企业私有工具  │  │
│  │ 16 toolsets│ │ MCP Server│ │ ArcPy Bridge │  │
│  └──────────┘ └──────────┘ └──────────────┘  │
│                    ↓                          │
│          ADK Agent (动态工具发现)              │
└──────────────────────────────────────────────┘
```

**子任务**:
1. **MCP Server 暴露**: 将现有 16 个 BaseToolset 暴露为标准 MCP Server 协议
2. **MCP Client 集成**: Agent 可动态连接外部 MCP Server（ArcGIS、QGIS 插件、遥感平台）
3. **工具市场 UI**: DataPanel 新增"工具"标签页，显示已连接/可用的 MCP 工具
4. **工具沙箱**: 外部 MCP 工具在隔离环境中执行，防止恶意工具影响系统

**技术选型**:
- MCP SDK (`@modelcontextprotocol/sdk`)
- ADK `McpToolset` 适配器
- WebSocket transport (支持 SSE 降级)

### 5.2 多模态输入/输出

**5.2.1 图像理解**
- 用户上传卫星影像/规划图纸/现场照片
- Gemini 2.5 Pro Vision 提取地物信息
- 自动标注 → GeoJSON 输出 → 地图渲染

**5.2.2 语音输入**
- Web Speech API 语音识别 → 文本 → 语义路由
- Gemini Audio 直接语音理解（野外巡查场景）
- 语音指令控制分析："放大到朝阳区"、"隐藏人口图层"

**5.2.3 PDF 智能解析**
- 直接上传规划文本 PDF → 结构化提取
- 表格提取 → DataFrame
- 坐标提取 → 自动定位
- 指标提取 → 对比分析

### 5.3 3D 空间可视化

**技术选型**: deck.gl (WebGL2) — 轻量、与 React 生态契合。

**3D 功能**:
- 3D 建筑体 (LoD2) + DEM 地形渲染
- 3D 专题图（柱状图叠加在地图上，数值映射到柱高）
- 飞行路线动画（时间序列数据播放）
- 日照分析可视化（建筑阴影模拟）

**集成方案**:
```
┌─────────────────────────────────────────────┐
│  MapPanel                                    │
│  ┌───────────┐  ┌───────────────────────┐   │
│  │ 2D (当前)  │  │ 3D (新增)             │   │
│  │ Leaflet.js │  │ deck.gl + MapLibre    │   │
│  │ GeoJSON    │  │ 复用 GeoJSON 数据流   │   │
│  └───────────┘  └───────────────────────┘   │
│  [2D] [3D] 切换按钮                          │
└─────────────────────────────────────────────┘
```

### 5.4 工作流编排 (Workflow Builder)

**问题**: 用户反复执行相同分析流程（每周数据治理、每月优化评估）。

**方案**:
1. **保存对话为工作流**: 将分析对话序列化为可重放的工作流模板
2. **可视化编辑器**: React Flow 拖拽式节点编辑器
   - 节点类型：数据输入、处理、分析、可视化、输出
   - 条件分支、循环、并行执行
3. **定时执行**: Cron 表达式调度 + Webhook 触发
4. **结果推送**: 通过企业微信/钉钉/飞书 Bot 推送分析结果

**新增文件**:
- `data_agent/workflow_engine.py` — 工作流执行引擎
- `frontend/src/components/WorkflowEditor.tsx` — React Flow 编辑器
- `data_agent/migrations/0xx_create_workflows.sql` — 工作流存储表

### 5.5 高级分析引擎

**5.5.1 时空预测**
- 地理加权回归 (GWR / GTWR)
- 基于历史数据的空间趋势预测
- 自动特征工程 + 模型选择

**5.5.2 场景模拟**
- "如果在 A 点建工厂，对周边土地价值的影响？"
- Monte Carlo 模拟 + 空间溢出效应模型
- 多方案对比可视化

**5.5.3 网络分析**
- 基于 OSM 路网的等时圈分析 (isochrone)
- 最优路径规划 (Dijkstra / A*)
- 设施服务覆盖评估

## v6.0 — 协同智能 + 边缘计算

**目标**: 从单用户分析工具进化为多用户协同决策平台。
**周期**: 3-6 月

### 6.1 实时协同编辑

- 多用户同时查看同一地图
- CRDT (Conflict-free Replicated Data Types) 冲突解决
- 光标共享 + 实时标注同步（类 Figma 模式）
- WebRTC 语音/视频通话集成（现场巡查协同）

### 6.2 知识图谱 (Spatial Knowledge Graph)

- Neo4j / Apache AGE 存储空间实体关系
- "这块地和哪些规划相关？" → 知识图谱推理
- 自动从历史分析中抽取实体关系
- 与语义层 (Semantic Layer) 深度集成

### 6.3 边缘部署 + 离线模式

- ONNX Runtime 本地推理（无需 Gemini API）
- Service Worker 离线缓存
- 野外调查场景：GPS + 拍照 + 本地分析 + 回连同步
- Progressive Web App (PWA) 支持

### 6.4 数据连接器生态

| 连接器 | 协议 | 用途 |
|--------|------|------|
| WMS/WFS/WMTS | OGC 标准 | 直连地图服务 |
| ArcGIS Online | REST API | ESRI 生态对接 |
| Google Earth Engine | Python API | 遥感大数据分析 |
| Mapbox | Vector Tiles | 高性能底图 |
| 国土"一张图" | 政务接口 | 规划数据对接 |
| MQTT | IoT 协议 | 传感器实时接入 |

### 6.5 多 Agent 协作

- 多个 Agent 并行处理不同区域的数据
- Agent 间通过 ADK `transfer_to_agent` 链式推理
- 场景："分别分析 A/B/C 三个区县，然后汇总对比"
- 基于 ADK `ParallelAgent` 实现并发派发
