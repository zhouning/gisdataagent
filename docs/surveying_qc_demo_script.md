## 开场 (2 分钟)

### 背景说明

> "我们基于《上海市新型基础测绘协同质管和智能质控范式研究》课题需求，构建了一套测绘质检智能体系统。系统的核心理念是：**以 AI 编排平台为中枢，通过 MCP 协议连接各专业工具，实现质检流程从'人工抽检'向'全域智控'的跃迁**。"

### 架构一句话

> "平台采用三层架构：Data Agent 核心编排层负责工作流管理、规则审查、报告生成；4 个独立子系统通过 MCP/REST 提供 CV 视觉检测、CAD 解析、ArcGIS Pro 深度学习、参考数据服务；底层基础设施层处理内外网隔离和算力调度。"

## 演示环节一：缺陷分类体系 (2 分钟)

### 操作步骤

1. 打开 Data Agent 前端 → 点击 **数据面板 → 运维 → 质检** Tab
2. 切换到 **"缺陷分类"** 视图

### 展示要点

- **5 大缺陷类别**：格式错误 (FMT)、精度偏差 (PRE)、拓扑错误 (TOP)、信息缺失 (MIS)、规范性错误 (NRM)
- **30 个缺陷编码**：每个编码关联严重等级 (A/B/C)、适用成果类型、是否可自动修正
- **基于 GB/T 24356**：严重缺陷权重 12、较严重缺陷权重 4、一般缺陷权重 1
- **加权评分公式**：score = 100 - Σ(severity_weight × count) / total_items × 100

### 话术

> "缺陷分类法是整个质检体系的基石。每一个发现的问题都会被归入这 30 个编码之一，带有明确的严重等级。这使得质量评分不再是主观判断，而是可量化、可追溯的。"

---

## 演示环节二：质检工作流模板 (2 分钟)

### 操作步骤

1. 在质检 Tab 中切换到 **"质检模板"** 视图
2. 展示 3 个预置模板：
   - **标准质检流程** (5 步, SLA 60 分钟)
   - **快速质检** (2 步, SLA 15 分钟)
   - **完整质检流程** (7 步, SLA 120 分钟, 含人工复核)
3. 点击 **"创建工作流"** 按钮，从标准模板实例化

### 展示要点

- 每步有 **SLA 超时阈值**：超时自动记录 + 可配置重试
- 工作流支持 **DAG 并行执行**：独立步骤可同时运行
- 标准流程：数据接收 → 预处理 → 规则审查 → 精度核验 → 报告生成

### 话术

> "我们预置了三种质检模板，覆盖日常复查到正式验收的不同场景。每一步都有 SLA 时间限制，超时会自动告警。最完整的流程还包含人工复核环节，确保重大缺陷不会被遗漏。"

## 演示环节三：端到端质检演示 (5 分钟) ⭐ 核心

### 数据说明

- **目标数据**：重庆市中心城区建筑物轮廓数据 (107,452 个面要素, WGS84)
- **参考数据**：重庆市中心城区历史文化街区数据 (20 个面要素, CGCS2000)

### 演示流程（实际运行结果）

**Step 1 — 数据格式校验**

> "首先自动校验数据格式。系统发现坐标系是 WGS84 (EPSG:4326)，根据国标应使用 CGCS2000 (EPSG:4490)，自动标记为规范性缺陷。"

| 检查项 | 结果 |
|--------|------|
| 记录数 | 107,452 |
| 坐标系 | EPSG:4326 (WGS84) |
| CRS 合规性 | ⚠ 需转换为 CGCS2000 |
| 几何类型 | Polygon / MultiPolygon |
| 字段 | Id, Floor, geometry |

**Step 2 — 几何拓扑检查**

> "第二步检查几何有效性和拓扑关系。在 10.7 万个要素中发现 417 个空几何和 416 个重复要素。几何有效性为 100%，没有自相交。"

| 检查项 | 结果 |
|--------|------|
| 无效几何 | 0 (100% 有效) |
| 空几何 | 417 (0.39%) |
| 重复几何 | 416 (0.39%) |

**Step 3 — 属性逻辑检查**

> "属性检查没有发现空值。层高分布从 1 层到 66 层，中位数 4 层，数值范围合理。"

| 统计项 | 值 |

| 最小层高 | 1 层 |
| 最大层高 | 66 层 |
| 平均层高 | 6.5 层 |

**Step 4 — 套合精度检查**

> "第四步是跨数据源套合检查。注意：建筑轮廓用的是 WGS84，历史文化街区用的是 CGCS2000——系统自动检测到 CRS 不一致并重投影后进行对比，发现 1,591 栋建筑位于历史文化街区范围内。"

| 检查项 | 结果 |
|--------|------|
| CRS 一致性 | ⚠ 不一致 (4326 vs 4490) |

## 演示环节四：报告自动生成 (1 分钟)

### 操作步骤

1. 在 Data Agent 聊天面板输入："根据刚才的质检结果生成标准质检报告"
2. 系统自动调用 `generate_qc_report()` 生成 Word 文档

### 报告内容

- 封面页：项目名称、检查单位、检查日期、检查依据
- 目录
- 8 个标准章节：项目概况、检查依据、数据审查结果、精度核验结果、缺陷统计、质量评分、整改建议、结论
- 动态表格 + 图表嵌入
- 标准排版（微软雅黑、A4、页眉页脚页码）

### 话术

> "报告按照 GB/T 24356 格式自动生成，包含封面、目录、8 个标准章节。所有检查数据自动填充到模板中，直接可用于正式验收。"

## 演示环节五：外部工具集成架构 (2 分钟)

### 展示 MCP Hub 配置

1. 打开 **数据面板 → 运维 → 追踪** Tab，或直接说明架构
2. 展示 4 个子系统通过 MCP 协议集成

### 架构图

```
Data Agent (核心编排)
     │ MCP协议
     ├── CV 视觉检测服务 (YOLO目标检测)
     ├── CAD/3D 解析服务 (ezdxf/trimesh)
     ├── ArcGIS Pro 双引擎
     │   ├── 基础引擎: 拓扑检查/空间分析/缓冲区/地图导出
     │   └── DL引擎: 目标检测/像素分类/变化检测/影像质量/超分辨率
     └── 参考数据服务 (控制点/基准信息)
```

### ArcGIS Pro 深度学习能力

- 本机已部署 ArcGIS Pro 3.6.2 + Deep Learning Libraries
- PyTorch 2.5.1 + arcgis.learn 2.4.2
- 可用模型类：SingleShotDetector, UnetClassifier, ChangeDetector, SuperResolution 等 40+
- 通过 subprocess 调用本机 ArcGIS Pro Python 环境，无需单独部署 GPU 服务

### 话术

> "我们通过 MCP 协议实现了松耦合集成。ArcGIS Pro 的深度学习能力可以直接被质检智能体调用——比如用 SingleShotDetector 做建筑物检测，用 ChangeDetector 做变化监测。这是'双引擎'架构：基础 GIS 操作走一个 Python 环境，深度学习推理走另一个带 PyTorch 的环境。"
