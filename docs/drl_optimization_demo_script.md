# 深度强化学习耕地空间布局优化 — 能力演示脚本

> 面向导师/评审的完整技术演示
>
> 演示时长：约 20 分钟
>
> 版本：v17.1 (2026-04-04)
>
> 前提：Data Agent 已启动 (`chainlit run data_agent/app.py -w`)，PostgreSQL + PostGIS 可用，测试数据已导入

---

## 开场 (2 分钟)

### 背景说明

> "耕地空间布局优化是国土空间规划和'三区三线'划定中的核心问题。传统方法依赖人工经验和简单的适宜性评价，难以在保持耕地面积总量平衡的前提下，同时优化坡度适宜性和空间连片度。我们基于深度强化学习（DRL）构建了一套自动化的空间布局优化系统——智能体通过'成对置换'策略，将高坡度耕地转换为林地、将低坡度林地转换为耕地，在维持面积平衡的约束下最小化耕地平均坡度、最大化耕地连片度。"

### 技术架构一句话

> "系统采用 MaskablePPO（可屏蔽动作空间的近端策略优化）算法，基于自定义 Gymnasium 环境运行。每个地块构建 6 维特征向量（坡度、类型、邻域比、邻域均坡、面积、坡度偏差），加上 8 维全局状态特征，构成观测空间。奖励函数由三项组成：坡度改善奖励、连片度改善奖励、面积偏差惩罚，再加上成对置换完成奖励。模型权重预训练在 `scorer_weights_v7.pt` 中，支持 3 种内置场景模板和自定义权重。"

## 演示环节一：数据准备与探查 (3 分钟)

### 场景设定

使用一份包含耕地（旱地、水田）和林地（有林地、果园）的地块矢量数据，数据包含 `DLMC`（地类名称）和 `Slope`（坡度）字段。数据可以从 PostGIS 数据库导出，也可以直接上传 Shapefile。

### 操作步骤

1. 登录 Data Agent 前端 (`localhost:8000`)，使用 `admin` / `admin123`
2. **方式 A — 数据库导出**：在聊天面板输入：
   > **"对 xx_parcels 表进行耕地空间布局优化分析"**
3. **方式 B — 文件上传**：将 Shapefile（.shp + .dbf + .shx + .prj）打包为 ZIP 上传，然后输入：
   > **"对上传的数据进行耕地空间布局优化"**
4. 系统自动分类意图为 `optimization`，进入优化管道
5. DataExploration 智能体自动执行：
   - 导出数据为本地 Shapefile
   - 调用 `describe_geodataframe` 进行数据画像
   - 输出数据文件路径和字段概要

### 展示要点

- **意图路由**：系统通过 Gemini 2.0 Flash 自动识别"布局优化"意图，路由到 Optimization Pipeline
- **RBAC 控制**：viewer 角色会被拒绝，需要 analyst 或 admin 权限
- **数据画像**：自动报告地块总数、字段列表、DLMC 值域分布
- **关键字段检查**：`DLMC`（地类名称）和 `Slope`（坡度）两个字段必须存在

### 话术

> "系统采用语义意图路由——我只需要说'布局优化'，Gemini Flash 路由器就会自动识别意图并分派到优化管道。管道的第一步是数据准备：自动导出数据、画像、确认关键字段。注意这里不需要手动筛选耕地或林地——DRL 模型内部已经定义了地类映射：DLMC 为'旱地'或'水田'归类为耕地，'有林地'或'果园'归类为林地，其他地类自动忽略。"

## 演示环节二：DRL 布局优化核心流程 (5 分钟) ⭐ 核心

### 场景设定

数据准备完成后，DataAnalysis 智能体自动调用 `drl_model` 工具执行深度强化学习优化。

### 操作步骤

1. 观察聊天面板中 DataAnalysis 智能体的执行过程
2. 系统自动执行以下步骤：
   - 加载 Shapefile，构建空间邻接图（Rook contiguity）
   - 初始化 `LandUseOptEnv` Gymnasium 环境
   - 加载预训练 MaskablePPO 模型权重 (`scorer_weights_v7.pt`)
   - 执行 200 步推理（每步选择一个地块进行类型转换）
   - 输出优化后的 Shapefile 和对比图
3. 观察输出结果：
   - 优化统计：转换次数、成对置换次数、耕地净变化
   - 优化前后坡度对比
   - 结果文件路径

### 展示要点

- **环境构建**：
  | 参数 | 值 |
  |------|----|
  | 每地块特征维度 | K=6 (坡度、类型、邻域耕地比、邻域均坡、面积、坡度偏差) |
  | 全局特征维度 | G=8 (均坡、连片度、面积偏差、进度等) |
  | 动作空间 | Discrete(可交换地块数) |
  | 最大步数 | 200 |

- **奖励函数设计** (v7 核心改进)：
  ```
  reward = slope_weight × Δ坡度 + contiguity_weight × Δ连片度 
           - balance_weight × (面积偏差)² + pair_bonus × 配对完成
  ```
  | 权重参数 | 默认值 | 含义 |
  |----------|--------|------|
  | slope_weight | 1000 | 坡度改善奖励权重 |
  | contiguity_weight | 500 | 连片度改善奖励权重 |
  | balance_weight | 500 | 面积偏差惩罚权重 (v7: 从 100,000 降至 500) |
  | pair_bonus | 1.0 | 成对置换完成奖励 (v7: 从 0.5 提升) |

- **成对置换策略**：
  - 将高坡度耕地 → 转为林地（减少耕地平均坡度）
  - 将低坡度林地 → 转为耕地（补充耕地面积）
  - 当耕地数量恢复到初始值时，给予 pair_bonus 奖励
  - 模型学会了"一减一加"的配对策略，而非单方向转换

- **动作屏蔽 (Action Masking)**：
  - 只允许操作耕地/林地地块，忽略其他地类
  - 已转换过的地块不可再次操作（防止撤销）
  - 保证智能体只在有效动作空间中决策

### 话术

> "现在进入核心环节——DRL 优化推理。这里有几个关键技术点值得注意：
>
> 第一，**v7 版本的奖励函数经过了重大调优**。v6 的面积偏差惩罚权重高达 100,000，导致随机探索阶段每个 episode 的总惩罚达到 -1,243，彻底淹没了梯度信号。v7 将其降到 500，随机探索惩罚降到约 6.3，同时将配对奖励从 0.5 提升到 1.0，使得配对策略的收益（约 0.7/步）明显优于单方向转换（约 0.2/步），模型因此学会了成对置换。
>
> 第二，**动作屏蔽**确保模型只在合法动作空间中决策——已转换的地块不可再操作，非耕地/林地地块不可选择。这比传统的惩罚式约束更高效，直接从动作空间中去除无效选项。
>
> 第三，**增量式指标更新**——每次地块转换后，坡度和连片度等指标通过增量更新计算，而不是全量重算，保证了推理效率。"

## 演示环节五：多目标 Pareto 前沿分析 (3 分钟) ⭐ 高级

### 场景设定

当用户关心"坡度最优"和"连片度最优"之间的权衡时，可以使用多目标优化生成 Pareto 前沿。

### 操作步骤

1. 在聊天面板输入：
   > **"对这份数据进行多目标布局优化，分析坡度和连片度的权衡"**
2. 系统调用 `drl_multi_objective` 工具
3. 内部执行 NSGA-II 多目标进化算法：
   - 种群大小=20，进化代数=10
   - 3 个优化目标：坡度分数、连片度分数、面积平衡分数
   - 非支配排序 + 拥挤度距离选择
4. 输出 Pareto 前沿散点图 (`pareto_frontier_xxx.png`)
5. 展示前沿解集及其目标值

### 展示要点

- **三维目标空间**：
  | 目标 | 方向 | 含义 |
  |------|------|------|
  | slope_score | 最大化 | 1 - (优化后均坡 / 初始均坡)，越高越好 |
  | contiguity_score | 最大化 | 优化后连片度 / 初始连片度，越高越好 |
  | area_balance | 最大化 | 1 - 面积偏差比，1.0 为完美平衡 |

- **NSGA-II 进化算法**：
  - 随机生成 20 组不同的 (slope_w, cont_w, balance_w) 权重组合
  - 每组权重运行一次 DRL 优化
  - 使用非支配排序（fast non-dominated sort）筛选 Pareto 前沿
  - 拥挤度距离维持解集多样性

- **Pareto 图解读**：
  - X 轴：坡度分数（越高 = 坡度优化越好）
  - Y 轴：连片度分数（越高 = 连片度越好）
  - 气泡大小：面积平衡分数
  - 前沿上的每个点代表一个"不被其他方案支配"的最优解

### 话术

> "在实际规划中，坡度和连片度往往是相互矛盾的目标——极端追求低坡度可能导致耕地碎片化，反之追求连片可能保留一些高坡度地块。Pareto 前沿分析可以帮助决策者理解这种权衡。我们使用 NSGA-II 进化算法，自动搜索多组权重组合下的最优解，通过非支配排序筛选出 Pareto 前沿。前沿上的每个点都是一个'不可改进'的方案——你无法在不牺牲一个目标的前提下改善另一个。决策者可以根据实际需求，在前沿上选择最适合的权衡点。"

## 演示环节六：自定义场景与参数调优 (3 分钟)

### 场景设定

演示 DRL 系统的可扩展性——除了默认的耕地优化场景，还支持城市绿地、设施选址等场景模板，以及自定义权重。

### 操作步骤

1. 在聊天面板输入：
   > **"使用城市绿地布局场景对数据进行优化"**
2. 系统使用 `urban_green_space` 场景模板，权重自动调整
3. 对比不同场景的效果差异

### 展示要点

- **3 种内置场景模板**：

  | 场景 ID | 名称 | 源类型 | 目标类型 | 坡度/连片/平衡 权重 |
  |---------|------|--------|----------|---------------------|
  | `farmland_optimization` | 耕地布局优化 | 旱地, 水田 | 果园, 有林地 | 1000 / 500 / 500 |
  | `urban_green_space` | 城市绿地布局 | 绿地, 公园, 草地 | 建设用地, 硬化地面 | 200 / 1000 / 800 |
  | `facility_siting` | 设施选址优化 | 公共服务设施 | 居住用地, 商业用地 | 300 / 800 / 600 |

- **权重含义差异**：
  - 耕地优化：坡度权重最高 (1000)，因为坡度是耕地适宜性核心因素
  - 城市绿地：连片度权重最高 (1000)，因为绿地连通性影响可达性
  - 设施选址：平衡权重和连片度较高，因为服务覆盖需要均匀分布

- **自定义权重**：
  > **"用坡度权重 1500、连片权重 300 重新优化"**
  - 支持运行时覆盖任意权重参数
  - 方便做敏感性分析

### 话术

> "DRL 系统的一个重要设计目标是可扩展性。我们预定义了 3 种场景模板，每种场景有不同的源/目标地类映射和权重配置。例如，耕地优化场景把坡度权重设为 1000——因为坡度是决定耕地适宜性的核心指标；而城市绿地场景把连片度权重设为 1000——因为绿地的生态效益和可达性取决于连通性。用户还可以直接指定自定义权重，做参数敏感性分析。整个场景模板机制基于 `DRLScenario` 类，添加新场景只需要定义地类映射和权重配置。"

## 演示环节七：World Model 增强 (选讲, 2 分钟) 🔬 前沿

### 场景设定

展示 Dreamer-style 世界模型与 DRL 的集成——AlphaEarth 世界模型提供未来 LULC 变化预测，作为辅助奖励信号注入 DRL 优化。

### 操作步骤

1. 在聊天面板输入：
   > **"使用世界模型增强的 DRL 优化分析这份数据，预测未来 3 年的变化"**
2. 系统调用 `dreamer_optimize` 工具
3. 参数说明：
   - `year`: 基准年份 (如 2023)
   - `look_ahead_years`: 世界模型预测年数 (如 3)
   - `aux_reward_weight`: 辅助奖励权重 (0.0-1.0，如 0.1)

### 展示要点

- **集成架构**：
  - DRL Agent 每隔 K 步调用 AlphaEarth 世界模型
  - 世界模型预测当前方案在未来 N 年的 LULC 变化趋势
  - 预测结果转化为辅助奖励，注入 DRL 奖励函数
  - 最终奖励 = 原始奖励 + α × 世界模型辅助奖励

- **组件**：
  - `ParcelEmbeddingMapper`: 地块嵌入向量映射器
  - `ActionToScenarioEncoder`: 动作序列编码为情景向量
  - `DreamerEnv`: 包装原始 LandUseOptEnv，注入 look-ahead 奖励

### 话术

> "这是技术前沿——Dreamer-style 世界模型与 DRL 的集成。传统 DRL 只看到当前步的即时奖励，而世界模型可以'看到未来'：预测当前优化方案在 3 年后的土地覆被变化趋势。如果某个置换方案虽然短期坡度改善大，但世界模型预测 3 年后该区域会因生态退化而不适宜耕作，辅助奖励就会降低该方案的吸引力。这让 DRL 智能体具备了长期视角。目前这是技术预览阶段，AlphaEarth 世界模型的嵌入精度还在优化中。"

## 总结 (2 分钟)

### 核心指标

| 指标 | 数值 |
|------|------|
| DRL 算法 | MaskablePPO (sb3_contrib) |
| 环境版本 | v7 (调优后: 降惩罚 + 增配对奖励 + 无早停) |
| 预训练权重 | scorer_weights_v7.pt |
| 特征维度 | K=6 (逐地块) + G=8 (全局) |
| 场景模板 | 3 种 (耕地 / 绿地 / 选址) |
| 多目标算法 | NSGA-II (非支配排序 + 拥挤度) |
| World Model | AlphaEarth + DreamerEnv (Tech Preview) |
| 管道架构 | 5 Agent 顺序管道 (Exploration → Processing → Analysis → Visualization → Summary) |
| 质检闭环 | QualityChecker 自动校验 + 重试循环 |

### 技术创新点

> "DRL 耕地布局优化系统有 4 个核心技术创新：
>
> 1. **成对置换策略 + 动作屏蔽**——通过 MaskablePPO 的 action masking 机制，智能体只能选择合法地块进行操作，配合 pair_bonus 奖励设计，模型自发学会了'一出一入'的成对置换策略，天然保持面积平衡；
>
> 2. **增量式指标更新**——每次地块转换后，坡度和连片度等核心指标通过增量方式更新（仅更新受影响的邻域地块），避免全量重算，将单步计算复杂度从 O(N) 降到 O(K)；
>
> 3. **多场景模板 + NSGA-II 多目标优化**——DRLScenario 模板机制支持多种规划场景，NSGA-II 提供多目标权衡分析，帮助决策者在 Pareto 前沿上选择最优方案；
>
> 4. **World Model 前瞻增强**——Dreamer-style 架构将 AlphaEarth 世界模型的长期预测注入 DRL 奖励函数，使智能体具备跨时间尺度的决策能力。"

### 与传统方法对比

| 维度 | 传统方法 | 本系统 |
|------|----------|--------|
| 优化方法 | 适宜性评价 + 人工调整 | MaskablePPO 深度强化学习 |
| 约束处理 | 事后校验 | 动作屏蔽 (action masking) 内嵌约束 |
| 目标权衡 | 单目标或加权求和 | NSGA-II Pareto 前沿 |
| 面积平衡 | 手动调配 | 成对置换 + 二次惩罚项 |
| 时间视角 | 仅当前时刻 | World Model 未来 1-10 年前瞻 |
| 交互方式 | GIS 软件操作 | 自然语言对话 → AI 自动执行 |
| 报告输出 | 手动编写 | 多智能体自动生成决策报告 |

### 下一步

> "后续的技术演进方向：
> - **更大规模数据支持**：当前环境内存加载，计划支持 PostGIS 下推 + 分块优化
> - **在线训练**：`train_drl_model` 工具已支持用户数据在线训练，未来可结合 GPU 加速
> - **更多场景**：交通网络优化、公共服务设施选址等场景的预训练权重
> - **World Model 精度**：提升 AlphaEarth 嵌入的时空分辨率"
