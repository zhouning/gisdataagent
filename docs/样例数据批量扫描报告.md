## 3. 关键发现

### 3.1 语义等价库覆盖不足

以下源数据字段未匹配，但实际上有明确的语义关系，需要补充到语义等价库：

| 源字段 | 应匹配到 | 说明 |
|--------|---------|------|
| DLDM | DLBM | 旧版字段名"地类代码" vs 新版"地类编码" |
| XZDLDM | — | "现状地类代码"，可能对应 DLBM 的一个变体 |
| XZDLMC | — | "现状地类名称"，对应 DLMC 的变体 |
| PDJB | GDPDJB | "坡度级别" vs "耕地坡度级别" |
| TKXS | KCXS | "田坎系数" vs "扣除地类系数"（同一概念不同命名） |
| TKMJ | KCMJ | "田坎面积" vs "扣除地类面积" |
| XZDWMJ | — | "线状地物面积"，新标准中无独立字段 |
| LXDWMJ | — | "路线地物面积"，新标准中无独立字段 |
| SM | BZ/MSSM | "说明" vs "备注"/"描述说明" |

### 3.2 新旧标准字段差异

源数据来自**村规划成果**（约 2015-2020 年期间），与当前标准（2024-2025 版）存在版本差异：
- 旧版使用 DLDM（地类代码），新版使用 DLBM（地类编码）
- 旧版使用 TKXS/TKMJ（田坎），新版使用 KCXS/KCMJ（扣除地类）
- 新版增加了大量条件必填字段（图斑细化、种植属性、飞入地标识等）

### 3.3 线状地物无独立标准表

JQLXDW（线状地物）和 JQXZDW（现状地物）在当前标准中没有独立的属性表定义。
线状地物的宽度信息在 DLTB 标准的 XZDWKD 字段中体现（注 9）。
这意味着数据治理时需要将线状地物信息合并到地类图斑中。
