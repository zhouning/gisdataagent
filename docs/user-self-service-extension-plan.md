# Data Agent 用户自助式能力扩展方案

## 1. 核心理念：从“硬编码”向“插件化”演进
为了满足用户不断增长的定制化需求，Data Agent 必须允许用户在不修改系统核心代码的前提下，动态注入**工具（Tools）**、**领域专家（Skills）**和**业务逻辑（Workflows）**。本方案基于 Google ADK 架构，提出了三个层级的渐进式扩展路径。

## 2. 三层扩展架构设计

### 2.1 第一层：基于 MCP 协议的工具扩展（即刻可用）
这是目前最安全、解耦最彻底的扩展方式。
- **技术实现**：利用 Model Context Protocol (MCP) 工业标准。
- **扩展机制**：用户可自行开发并运行独立的 MCP Server（支持 Python, Node.js 等），封装自有的私有 API 或专业算法。
- **自助方式**：用户在系统界面或 `mcp_servers.yaml` 配置文件中填入自定义服务器的 URL（SSE）或命令（Stdio）。
- **优势**：计算资源隔离，用户的自定义代码即使崩溃也不会影响主系统性能。

### 2.2 第二层：自定义领域专家 Skills（提示词与 RAG）
允许用户注入特定的行业思考逻辑。
- **技术实现**：将 Prompt 模板和 RAG 知识库配置从硬编码改为数据库持久化存储。
- **扩展机制**：
    - **自定义指令**：用户在 UI 面板录入“Skill 名称”及“系统提示词（System Prompt）”，定义 Agent 的专家身份与分析规范。
    - **挂载私有知识库**：用户上传行业 PDF 文档，系统动态创建 RAG 引擎（如 Vertex AI Search 工具）并挂载给该会话。
- **自助方式**：前端提供“我的技能库”管理面板。

### 2.3 第三层：可视化多智能体工作流（低代码编排）
允许用户根据业务需求重新定义 Agent 之间的协作关系。
- **技术实现**：打通前端 **React Flow** 画布与后端 **ADK SequentialAgent / LoopAgent** 的动态实例化逻辑。
- **扩展机制**：
    - 将内置的原子智能体（Explorer, Processor, Analyzer 等）和工具集抽象为画布节点。
    - 用户通过拖拽连线，自定义数据流向和逻辑判定节点（如：`数据探查 -> 用户自定义MCP工具 -> 质量审核循环 -> 生成报告`）。
- **自助方式**：全开放的 Web 流程编辑器。

## 3. 落地实施建议（Roadmap）

1.  **阶段一：完善 MCP 市场 (Short-term)**
    - 在前端 DataPanel 中实现“添加自定义 MCP 服务器”的 UI 交互。
    - 开放 API 支持动态刷新 `McpHubManager` 的连接清单。
2.  **阶段二：开放“自定义 Skill”配置 (Mid-term)**
    - 实现 `agent_custom_skills` 数据库表，支持多租户隔离的提示词管理。
    - 允许用户在对话时通过 `@专家名称` 的方式唤起自定义技能。
3.  **阶段三：多智能体动态装配引擎 (Long-term)**
    - 重构 `workflow_engine.py`，支持直接解析 React Flow 生成的 JSON 拓扑图并转化为 ADK 运行时的 Agent 树状结构。
