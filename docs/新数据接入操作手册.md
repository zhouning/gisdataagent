# 新数据接入操作手册（NL2Semantic2SQL 冷启动流程）

## 目标

本手册用于指导管理员将一套全新的 PostgreSQL/PostGIS 数据接入 GIS Data Agent，使其经过扫描、语义草稿、人工审核、冷启动验证和激活后，进入可供 NL2SQL 使用的 active 语义层。

流程：

`scan -> draft -> review -> validate -> activate`

---

## 前提条件

1. 新数据已导入 PostgreSQL/PostGIS
2. 表位于 `public` schema（当前默认）
3. geometry 列已正确注册到 `geometry_columns`
4. 管理员账号可访问系统前端或 API

---

## 方式一：通过前端界面接入

### 第 1 步：扫描新表

1. 进入 DataPanel
2. 打开 **“接入”** Tab
3. 点击 **“扫描新表”**

系统会：
- 扫描 `public` schema 下所有非 `agent_*` 表
- 生成 dataset profile
- 记录 geometry 类型、行数、字段、样例值、风险标签

### 第 2 步：生成语义草稿

对目标表点击 **“生成草稿”**。

系统会：
- 自动推断 `display_name`
- 为字段生成 `aliases`
- 识别 `semantic_domain`
- 推断 `join_candidates`
- 标记风险字段

### 第 3 步：验证

当表状态为 `drafted` 或 `reviewed` 时，点击 **“验证”**。

系统会自动生成冷启动题，覆盖：
- 过滤
- 聚合
- Top-K
- Security Rejection
- Anti-Illusion

只有 `eval_score >= 0.8` 才建议激活。

### 第 4 步：激活

当验证通过后，点击 **“激活”**。

系统会：
- 将草稿写入 `agent_semantic_sources`
- 将字段语义写入 `agent_semantic_registry`
- 记录 activation 版本
- 让该表进入 NL2SQL active 层

### 第 5 步：回滚（如有需要）

如果激活后效果不佳，可点击 **“回滚”**。

系统会：
- 回退到上一个 active 版本
- 若无历史版本，则将该表从 production semantic 层移除

## 方式二：通过 REST API 接入

### 1. 扫描

```bash
curl -X POST http://localhost:8000/api/intake/scan \
  -H "Content-Type: application/json" \
  -d '{"schema":"public"}'
```

### 2. 查看 profiles

```bash
curl http://localhost:8000/api/intake/profiles
```

### 3. 生成 draft

```bash
curl -X POST http://localhost:8000/api/intake/<dataset_id>/draft
```

### 4. 审核（示例）

```bash
curl -X POST http://localhost:8000/api/intake/<draft_id>/review \
  -H "Content-Type: application/json" \
  -d '{
    "blocked_columns": ["password", "secret_token"],
    "approved_joins": ["other_table"],
    "notes": "确认 geometry 与主键候选正确"
  }'
```

### 5. 验证

```bash
curl -X POST http://localhost:8000/api/intake/<dataset_id>/validate
```

### 6. 激活

```bash
curl -X POST http://localhost:8000/api/intake/<draft_id>/activate \
  -H "Content-Type: application/json" \
  -d '{"eval_score": 0.85}'
```

### 7. 回滚

```bash
curl -X POST http://localhost:8000/api/intake/<dataset_id>/rollback
```

---

## 激活门槛

默认规则：

- `eval_score >= 0.8` 才允许激活
- 若确需人工强制上线，可调用 activate 时传 `force=true`（仅建议管理员在充分了解风险时使用）

---

## 风险提示

以下情况建议不要直接激活：

1. **多 geometry 列** 且未确认默认空间列
2. **弱 join 关系**，系统自动推断不可靠
3. **敏感字段**（PII、token、secret）未屏蔽
4. **超大表** 尚未确认采样/限流策略
5. `eval_score < 0.8`

---

## 上线后的观察指标

激活后建议持续观察：

- NL2SQL 成功率
- 自纠错触发率
- few-shot 命中率
- 幻觉率 / 误拒绝率
- 用户人工纠错反馈

如果指标恶化，优先回滚到上一个 active 版本。

---

## 与现有系统的关系

- 运行时仍复用现有：
  - `nl2sql_grounding.py`
  - `sql_postprocessor.py`
  - `nl2sql_executor.py`
- 冷启动接入只负责决定：
  - 哪些表进入 active semantic 层
  - 哪些字段可用
  - 哪些 few-shot 在当前域优先检索

---

## 建议的人工验证顺序

当你准备好新数据后，建议这样验证：

1. 扫描表
2. 生成草稿
3. 审核关键字段与 join
4. 跑 validate
5. 激活
6. 人工问 5–10 个代表性问题
7. 若表现稳定，再扩大开放范围
