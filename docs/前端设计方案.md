# GIS Data Agent — 前端设计方案

> 日期：2026-04-07（v2 更新：三面板布局 + 地图常驻）
>
> 定位：独立界面，React 技术栈，复用原型分支组件
>
> 设计原则：对话驱动 + 地图验证 + 结构化详情，三者在同一界面中协同

---

## 1. 整体布局

### 1.1 三面板布局（对话 | 地图 | 工作面板）

```
┌──────────────────────────────────────────────────────────────────┐
│  GIS Data Agent — 时空数据智能治理          [步骤导航]       [用户] │
├──────────────┬──────────────────────┬────────────────────────────┤
│              │                      │                            │
│  对话面板     │     地图面板          │       工作面板              │
│  (左)        │     (中)             │       (右)                 │
│              │                      │                            │
│ AI 智能入口   │  空间数据验证         │  结构化操作与详情             │
│ · 分析过程    │  · 数据范围预览       │  · 数据集列表/上传            │
│ · 自然语言    │  · 按字段值着色       │  · 字段匹配表格              │
│ · 推荐解释    │  · 问题图斑高亮       │  · 差距报告详情              │
│ · 追问确认    │  · 质检错误定位       │  · 模型调整建议              │
│              │  · 治理前后对比       │  · 治理执行进度              │
│              │                      │  · 报告预览与导出             │
│              │                      │                            │
├──────────────┴──────────────────────┴────────────────────────────┤
│  状态栏：当前步骤 · 数据集名称 · 匹配率 · 底座连接状态               │
└──────────────────────────────────────────────────────────────────┘
```

**延续原型的三面板思路（Chat | Map | Data），但面板职责完全针对数据治理场景重新定义。**

面板宽度可拖拽调整（复用原型的 resizer 组件），默认比例约 25% | 40% | 35%。

### 1.2 三面板分工

| 对话面板（左） | 地图面板（中） | 工作面板（右） |
|-------------|-------------|-------------|
| AI 智能交互入口 | 空间数据的可视化验证 | 结构化数据和操作 |
| 自然语言对话 | 上传数据空间范围预览 | 表格、列表、进度条 |
| 分析过程流式展示 | 问题图斑空间定位 | 上传、选择、勾选、导出 |
| 推荐方案解释 | 质检错误标注 | 差距详情展开查看 |
| 追问和确认 | 治理前后对比 | 报告预览和下载 |

### 1.3 三面板联动

**对话 → 地图**：
- AI 说"发现 3 个拓扑错误" → 地图自动高亮这 3 个图斑位置
- AI 说"SJNF 字段缺失" → 地图将缺失该字段的图斑标红

**对话 → 工作面板**：
- AI 完成字段匹配 → 工作面板自动渲染匹配结果表格
- AI 输出调整建议 → 工作面板显示可勾选的建议列表

**工作面板 → 地图**：
- 用户在工作面板点击某条差距记录 → 地图定位到该图斑
- 用户在工作面板点击某个质检错误 → 地图缩放到错误位置

**工作面板 → 对话**：
- 用户上传数据 → 对话面板 AI 自动开始分析
- 用户勾选调整建议并确认 → 对话面板 AI 开始执行治理

**地图 → 工作面板**：
- 用户在地图上点击某个图斑 → 工作面板展示该图斑的字段详情和合规状态

## 2. 地图面板在每个步骤中的作用

### 步骤 1：上传数据
```
地图面板：
  · 渲染上传的 Shapefile/GeoJSON 的空间范围
  · 自动缩放到数据范围（fitBounds）
  · 底图叠加（天地图/高德）确认位置
  · 鼠标悬浮显示图斑基本信息（地类编码、面积）
  · 总工一眼能看到："这是璧山区福禄镇的数据"
```

### 步骤 2：数据画像
```
地图面板：
  · 按字段值着色渲染（如按地类编码 DLBM 分色）
  · 高亮空值记录：哪些图斑缺关键字段（半透明红色覆盖）
  · 标注几何异常：自相交、面积为0、坐标明显偏移
  · 右面板点击某条异常记录 → 地图自动定位并高亮该图斑
```

### 步骤 3：标准对照
```
地图面板：
  · 按合规状态着色渲染：
    🟢 绿色 = 完全合规
    🟡 黄色 = 部分合规（条件字段缺失）
    🔴 红色 = 严重不合规（必填字段缺失/值域越界）
  · 点击图斑 → 弹出该图斑的字段合规详情
  · 图例面板显示合规分布统计
```

### 步骤 4：模型推荐
```
地图面板：
  · 保持步骤3的合规着色
  · 当用户查看某个调整建议时，高亮受影响的图斑范围
  · 例如：选中"DLDM→DLBM映射"建议 → 地图高亮所有有DLDM值的图斑
```

### 步骤 5：治理执行（质检环节）
```
地图面板：
  · 质检错误空间定位：
    📍 拓扑错误（重叠/缝隙/自相交）→ 错误位置标记点+错误范围高亮
    🔶 属性错误（编码越界/面积不一致）→ 问题图斑高亮
  · 点击错误标记 → 对话面板 AI 解释错误原因和修复建议
  · 错误修复后标记变为灰色（已处理）
```

### 步骤 6：成果报告
```
地图面板：
  · 治理前后对比视图：
    方式A：左右分屏（左半治理前，右半治理后）
    方式B：切换按钮（治理前/治理后 toggle）
  · 治理前：红黄绿着色
  · 治理后：大部分变绿
  · 提供"截取当前视图"按钮 → 将地图截图嵌入 Word 报告
```

## 4. 报告中的地图截图

治理成果报告（Word）中需要包含地图截图，实现方式：

```
报告结构（增强版）：
  1. 概述
  2. 数据空间范围
     · [地图截图：数据集的空间范围总览]
  3. 治理前数据质量
     · 质量指标表格
     · [地图截图：按合规状态着色的治理前视图]
  4. 差距分析
     · 差距表格
  5. 治理方案
  6. 治理后数据质量
     · 质量指标对比表格
     · [地图截图：治理后视图——大部分变绿]
  7. 关键问题定位
     · [地图截图：质检错误的空间定位]
  8. 调整建议总结
  附录
```

技术实现：
- 前端使用 Leaflet 的 `leafletImage` 或 deck.gl 的 `deck.canvas.toDataURL()` 截取当前地图视图
- 截图上传到后端 → 嵌入 Word 报告（python-docx 的 `add_picture()`）
- 或前端直接将截图作为 base64 传给报告生成 API

## 8. 技术要点

### 8.1 三面板联动机制

```typescript
// 共享状态（Zustand store）
interface WorkspaceState {
  // 数据
  currentDataset: DatasetInfo | null;
  matchResult: FieldMatchResult[] | null;
  gapReport: GapReport | null;
  adviceList: AdjustmentAdvice[] | null;
  governanceStatus: GovernanceStatus | null;
  
  // 地图联动
  mapHighlightIds: string[];        // 高亮的图斑 ID 列表
  mapFocusFeatureId: string | null; // 定位到某个图斑
  mapColorMode: 'default' | 'compliance' | 'field_value' | 'errors';
  mapCompareMode: 'before' | 'after' | 'split';
  
  // 步骤
  currentStep: 1 | 2 | 3 | 4 | 5 | 6;
}

// 右面板点击差距项 → 更新 mapFocusFeatureId → 地图自动缩放
// 地图点击图斑 → 更新 selectedFeature → 右面板展示详情
// AI 完成分析 → 更新 matchResult + mapColorMode → 地图重新着色
```

### 8.2 地图渲染性能

- 数据量 < 5000 要素：直接 GeoJSON + Leaflet（足够覆盖村级数据）
- 数据量 5000-50000：FlatGeobuf 流式加载
- 数据量 > 50000：矢量切片（MVT）——复用原型的 tile_server.py

Phase 1 的演示数据（662 条图斑）用 GeoJSON 直接渲染即可。

### 8.3 UI 风格

延续原型的 Palantir 风格深色主题（Deep Intelligence 色系）：
- 背景：#0B0F19
- 地图合规着色：绿 #22C55E / 黄 #EAB308 / 红 #EF4444
- 字体：Inter + JetBrains Mono
- 图标：Lucide
