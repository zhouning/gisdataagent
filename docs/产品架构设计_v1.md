## 1. 系统定位

### 1.1 一句话定位

**GIS Data Agent 是时空数据治理平台的"大脑"——用领域知识驱动 AI Agent，让能力不足的实施人员也能完成数据治理项目交付。**

### 1.2 与底座的关系

```
┌────────────────────────────────────────────────────────────┐
│                   GIS Data Agent                            │
│                                                            │
│  ┌──────────┐  ┌──────────────┐  ┌───────────────────────┐ │
│  │ 前端     │  │ Agent 编排层  │  │ 平台级基础能力         │ │
│  │ 三面板UI │←→│ ADK Agent    │  │ · 认证/RBAC           │ │
│  │ 管理端   │  │ 多Agent协作   │  │ · 记忆/安全           │ │

│                ┌──────┴───────┐                            │
│                │ 知识管理层    │                            │
│                │ 语义/标准/   │                            │
```

### 1.3 价值差异

GIS Data Agent 和底座前端调用同一套 API，差异在于 API 调用**之前**（知识驱动的分析推理）和**之后**（自动生成报告）。详见 `docs/GIS_Data_Agent与底座前端的价值差异.md`。

### 2.2 Agent 编排层设计

#### 核心 Agent 定义

复用原型验证过的 ADK 模式：LlmAgent + SequentialAgent + LoopAgent + AgentTool。

```python
# intelligence/agent.py

from google.adk.agents import LlmAgent, SequentialAgent, LoopAgent
from google.adk.tools import AgentTool
from .tools import (
    analyze_fields_tool,
    check_compliance_tool,
    compare_model_tool,
    execute_governance_tool,
    generate_report_tool,
    render_map_tool,
    manage_knowledge_tool,
)
from data_agent.prompts import get_prompt

# --- 数据感知 Agent ---
data_profiling_agent = LlmAgent(
    name="DataProfiling",
    instruction=get_prompt("governance_v2", "data_profiling_instruction"),
    description="数据感知专家：识别字段含义、生成数据画像、发现质量问题",
    model="gemini-2.5-flash",
    output_key="data_profile",
    tools=[
        analyze_fields_tool,      # 语义等价库匹配
        render_map_tool,          # 在地图上渲染数据
    ],
)

# --- 标准对照 Agent ---
compliance_agent = LlmAgent(
    name="ComplianceCheck",
    instruction=get_prompt("governance_v2", "compliance_instruction"),
    description="标准对照专家：将数据现状与标准规则对照，生成差距报告",
    model="gemini-2.5-flash",
    output_key="gap_report",
    tools=[
        check_compliance_tool,    # 标准规则库对照
        render_map_tool,          # 合规状态着色
    ],
)

# --- 模型推荐 Agent ---
model_advisor_agent = LlmAgent(
    name="ModelAdvisor",
    instruction=get_prompt("governance_v2", "model_advisor_instruction"),
    description="模型推荐专家：对比数据现状与目标模型，推荐调整方案",
    model="gemini-2.5-flash",
    output_key="adjustment_advice",
    tools=[
        compare_model_tool,       # 数据模型库对比
    ],
)

# --- 治理执行 Agent ---
governance_executor_agent = LlmAgent(
    name="GovernanceExecutor",
    instruction=get_prompt("governance_v2", "executor_instruction"),
    description="治理执行专家：编排治理步骤，调用底座执行",
    model="gemini-2.5-flash",
    output_key="governance_result",
    tools=[
        execute_governance_tool,  # 底座 API 调用
        render_map_tool,          # 质检错误定位
    ],
)

# --- 质量审核 Agent（LoopAgent 模式，复用原型的 Generator-Critic 闭环）---
quality_reviewer_agent = LlmAgent(
    name="QualityReviewer",
    instruction=get_prompt("governance_v2", "reviewer_instruction"),
    description="质量审核员：检查治理结果是否达标，不达标则指导修正",
    model="gemini-2.5-flash",
    output_key="review_result",
)

governance_quality_loop = LoopAgent(
    name="GovernanceQualityLoop",
    sub_agents=[governance_executor_agent, quality_reviewer_agent],
    max_iterations=3,
)

# --- 报告生成 Agent ---
report_agent = LlmAgent(
    name="ReportGenerator",
    instruction=get_prompt("governance_v2", "report_instruction"),
    description="报告生成专家：汇总治理成果，生成可验收的 Word 报告",
    model="gemini-2.5-flash",
    output_key="governance_report",
    tools=[
        generate_report_tool,     # Word 报告生成
        render_map_tool,          # 地图截图嵌入报告
    ],
)

# --- 完整治理管线（SequentialAgent）---
governance_pipeline = SequentialAgent(
    name="GovernancePipeline",
    description="数据治理全流程：感知→对照→推荐→执行→审核→报告",
    sub_agents=[
        data_profiling_agent,
        compliance_agent,
        model_advisor_agent,
        governance_quality_loop,
        report_agent,
    ],
)

# --- 路由 Agent（顶层入口）---
# 用户可能不按步骤走，路由 Agent 判断意图后分发
router_agent = LlmAgent(
    name="GovernanceRouter",
    instruction=get_prompt("governance_v2", "router_instruction"),
    description="治理路由：判断用户意图，分发到对应的专家 Agent",
    model="gemini-2.0-flash",  # 路由用快速模型
    sub_agents=[governance_pipeline],
    tools=[
        # 子 Agent 也可以作为 Tool 按需调用（复用原型的 AgentTool 模式）
        AgentTool(agent=data_profiling_agent),
        AgentTool(agent=compliance_agent),
        AgentTool(agent=model_advisor_agent),
        AgentTool(agent=report_agent),
        manage_knowledge_tool,     # 知识库查询和管理
    ],
)
```

#### Agent 编排模式（复用原型验证过的）

| 模式 | 原型中的验证 | 产品中的应用 |
|------|------------|-------------|
| **SequentialAgent** | data_pipeline: Ingestion→Processing→Analysis→Viz→Summary | governance_pipeline: Profiling→Compliance→Advisor→Execute→Report |
| **ParallelAgent** | ParallelDataIngestion: Exploration ‖ SemanticPreFetch | 可用于：数据画像 ‖ 标准规则预加载 |
| **LoopAgent** | analysis_quality_loop: Analysis→QualityChecker→(repeat) | governance_quality_loop: Execute→Reviewer→(repeat if not pass) |
| **AgentTool** | knowledge_tool = AgentTool(agent=knowledge_agent) | 路由 Agent 按需调用子 Agent |
| **after_tool_callback** | _self_correction_after_tool（工具出错后自动重试） | 直接复用 |
| **output_key** | 每个 Agent 写入 session state 的特定 key | 治理 Agent 间通过 output_key 传递中间结果 |

### 3.5 地图面板在每个步骤中的用途

| 步骤 | 地图渲染内容 | 联动 |
|------|------------|------|

| 模型推荐 | 受影响图斑高亮 | 选中建议→高亮受影响范围 |
| 治理执行 | 质检错误空间定位（📍标记） | 点击标记→AI 解释错误 |
| 成果报告 | 治理前后对比（toggle/split） | 截图嵌入 Word 报告 |

## 6. 关键设计决策记录

| 决策 | 选择 | 理由 |
|------|------|------|
| 产品 vs 原型分支 | 独立分支，模式 A 集成 | 定位不同，避免互相干扰 |
| 前端技术栈 | React（复用原型） | 独立界面，不需和 Vue 底座统一 |
| Agent 框架 | ADK（复用原型） | 已验证 LlmAgent/Sequential/Loop/AgentTool 模式 |
| 知识管理 vs LLM | 确定性逻辑在知识层，LLM 做兜底推理 | 领域知识是壁垒，LLM 不可靠 |
| 底座对接 | REST API（路径 A） | API 现成，团队可控 |
| 地图 | Leaflet 常驻（复用原型 MapPanel） | 测绘客户必需，5/6 步骤用到 |
| 管理端 | 集成在同一前端（Tab 切换） | 演示时需要，减少维护成本 |
