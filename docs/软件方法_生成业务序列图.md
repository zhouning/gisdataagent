# 背景
我想用《软件方法》所讲述的方法学来开发系统，目前正在描述需要改进的现状业务流程。
# 任务
请从一段关于数据质检业务流程的原始文本提炼出业务流程，并使用序列图描述。
# 所输入信息的规范
一段原始文本。
# 要求输出信息的规范：
输出UML序列图，用PlantUML格式表达。在结尾附上你如何利用附件的参考资料上的知识提炼类和消息的详细思考过程。
# 约束和建议做法
我希望你按照以下思路来思考：
\*先分析所要求输出格式，确保满足要求。
\*识别流程描述中的人脑系统，建模为"业务工人" 构造型的类。
\*识别流程描述中的非人智能系统，建模为"业务实体" 构造型的类。
\*构造序列图，在序列图上安排所识别的业务工人、业务实体的实例。
\*识别流程描述中的行为，思考这个行为怎样能理解成"A请求B做某事"的形式，在序列图上添加A指向B的消息，消息名称为"做某事"。在识别行为的时候，要注意不要忽略人或非人智能系统自身思考或行动的行为。
